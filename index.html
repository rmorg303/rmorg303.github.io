Projects

Inverse Kinematics Animation System

  • Coded a robust inverse kinematics animation system in the game Garry's Mod using an in-game high-level language based on Lua named Expression 2.

  • Takes several user inputs and user-defined properties, such as:

    • Individual length of each limb.

    • Number of joints (for either plantigrade or digitigrade-type legs.)

    • Walking and running speeds.

    • Stride length for both walking and running states.

    • Crouch height.

    • Jump Height.

  • Uses a deterministic state machine to switch between different stats of movement (walking, running, crouching, jumping) and can be used to make walking robots in the game.

  • Runs efficiently, as Expression 2 is limited by tight constraints placed on operations per second.

  • Dynamic—the code will detect objects under its feet and adjust its stance accordingly.

  • Tech Stack:

    • Wiremod

      • Provides the editor for Expression 2.

    • Entity Core

      • Optional Wiremod addon that features functions which allow Expression 2 to interact with in-game entities.

    • Constraint Core

      • Optional Wiremod addon that features functions which bind in-game entities to each other in various ways.

Small-Group Polling Web Application

  • As a capstone project, I lead a group of programmers in creating an online polling application intended for small groups.

  • Core Features:

    • Could create an account if the user supplied a username, password, and valid email address. An email would be sent to the supplied email asynchronously along with a link containing a 24-character long randomly-generated code that expired after 30 minutes. The user would have to click that link to activate their account in the SQL-based database.

    • Using the same algorithm that generated the 24-character long code mentioned above, users could change their password via a link sent via email if they forgot their password. If they wanted to change it while logged in, they would have to enter their password on another page accessible via the user's dashboard.

    • Confirmed users could create their own small group and send invitations to others using their email addresses.

    • The group owner was able to promote other users to administrators, allowing them access to higher-level features, such as creating and publishing polls, or removing users from the group.

    • In the case that the group owner has to leave the group, they must promote a member of the group before departing.

    • The group owner may also delete the group, which also erases any related entries in the database.

    • Included a page that recorded the history of polls voted on by the group as well as their final results.

  • Tech Stack:

    • Java

      • As the primary language used at Franklin University, it ensured that all programmers were familiar with the syntax, data structures, and logic.

    • Spring Boot

      • Easy-to-use and manage, also included a dependency to send email.

    • H2 Database

      • Chosen for its speed and compatibility with JDBC

Mobile Weather Application

  • For my mobile application development course final project, I chose to create a weather application for users who frequent other parts of the country, such as service call engineers employed by medical technology companies.

  • Core Features:

    • Used Google Maps to display a world map and allow the creation of pins on locations the user would regularly visit.

    • Interacted with the OpenWeatherMap API to retrieve the current weather forecast, including:

      • General Conditions

        • A general, natural language description of the location.

        • The current actual and apparent ("feels-like") temperature.

        • The predicted high temperature.

        • The predicted low temperature.

        • The rain volume.

        • The visibility range.

      • Atmospheric Conditions

        • The humidity.

        • The cloud coverage.

        • The air pressure.

      • Wind Conditions

        • The wind speed.

        • The wind direction.

    • The ability to customize the app, such as enabling dark mode or swapping between imperial, metric, or standard measurements.

  • Tech Stack:

    • Flutter

      • Google's own open-source SDK that made it possible to develop simultaneously for Apple, Android, Microsoft, and Linux devices all at once.

    • Google Maps API

      • Allowed the user to create pins on the map, which would be used to retrieve the weather using the location of the pin.

    • OpenWeatherMap API

      • By supplying it with the latitude and longitude from the Google Maps pins, the weather conditions of the region could be obtained.

About Me

I am a passionate sort of person, interested in using my skills and knowledge to help people and deliver quality results. Complex structures and logic are of special interest to me, whether it is part of the structure of a program or it is the structure of a story or plot. I love to read, too, and am currently making my way through George R.R. Martin's A Song of Ice and Fire series. I am currently reading the second book, A Clash of Kings. Afterwards, I intend to read through Gene Wolfe's Solar Cycle, considered a staple of the "dying earth" genre, starting with his 1980 novel Shadow of the Torturer. My favorite book, one that has inspired much of the media that I enjoy today, is the 1972 novel Roadside Picnic by Arkady and Boris Strugatsky.

I also, unsurprisingly, have a deep love of video games. Games with great detail and deep mechanics tend to hold my attention and captivation the most. My favorite game is Bethesda Softworks' The Elder Scrolls III: Morrowind, released back in 2002, in part because of its meticulous world-building and alien society/philosophy, which sets it apart from its many other Tolkienesque fantasy worlds. Another notable favorite of mine is Bungie's 2001 title Halo: Combat Evolved, along with the studio's prior FPS series, Marathon, which began in 1994 and ended in 1996 (although a reboot, simply named Marathon, came out in 2026). Halo particularly had a predictable-yet-intelligent AI system and a minimalist weapon sandbox that made gunfights dynamic and the tools the player could use unique in their applications, which I like to think has had some effect on the way I program: to keep what works well and eliminate the useless bloat without sacrificing any part of the experience. Recently, I have developed a fascination with colony-building games, which feature wonderfully complex and interconnected systems that make or break the survival of your settlement, such as managing food stores, weapons and armor production, and cash flow. In particular, I very much enjoy the still-in-development 2006 title Dwarf Fortress by Bay 12 Games and the 2025 game Farthest Frontier by Crate Entertainment.
